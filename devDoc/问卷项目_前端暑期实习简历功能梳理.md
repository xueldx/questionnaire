# 问卷小筑 · 功能回顾总入口

> 本文件是 devDoc 知识库的总入口和回顾任务看板。
> 所有功能的复盘状态、文档双链、简历价值评级都在此维护。
> 编写和维护规则见 [[AGENTS]]（项目根目录），代码导航入口见 [[ARCHITECTURE]]（项目根目录）。

---

## 项目总目标

通过系统性地回顾已实现功能和学习新功能代码，达成：

1. 重新理解或新学习每个功能的关键代码和设计思路
2. 沉淀为可复习、可支持面试表达的技术文档
3. 补足功能背后的前端基础概念、知识点、八股和教学例子
4. 形成对当前方案的工程判断力，并沉淀可落地的优化思路
5. 将高价值功能的技术描述同步到 [[claude-简历母版]]
6. 每个功能稳定产出四份文档：概念学习 + 学习拆解 + 功能复盘  + 方案评审

### 每个功能对应的文档

- 概念学习文档：`概念学习-{功能名}.md`，帮助补基础、补知识点、补八股
- 学习拆解文档：`学习拆解-{功能名}.md`，帮助重新看懂代码
- 功能复盘文档：`功能复盘-{功能名}.md`，帮助准备简历和项目表达
- 方案评审文档：`方案评审-{功能名}.md`，帮助形成工程判断力和优化表达能力

### 固定执行顺序

1. 输出 `概念学习-{当前功能}.md`
2. 输出 `学习拆解-{当前功能}.md`
3. 输出 `功能复盘-{当前功能}.md`
4. 输出 `方案评审-{当前功能}.md`
5. 更新总入口 / 导航 / 双链
6. 如果有必要，同步更新 [[claude-简历母版]]

| 状态 | 含义 |

### 功能条目补充字段

- `入口文件`：第一次进入该功能时，最应该先打开的页面 / 路由 / controller / hook
- `核心文件`：3-8 个关键文件，按职责列出，不要把整个目录都抄进去
- 如果当前还没梳理到这一步，可以先写“待补”
- 正式回顾前，建议先看根目录 `ARCHITECTURE.md` 做代码导航，再回到源码核验

## 后续新增功能登记方式

当有新功能需要纳入回顾体系时：

1. 在上方对应批次中新增一个功能条目，填写完整的属性表格
2. 创建四个双链占位（学习拆解 + 功能复盘 + 概念学习 + 方案评审），标注“待创建”
3. 按 [[AGENTS]]（项目根目录）中的命名规范命名文档
4. 评估简历价值和优先级，插入到合适的位置
5. 如果已经完成入口梳理，额外补充 `入口文件` 和 `核心文件`
6. 如果是待开发功能，在“证据是否充分”中标注“待开发”
