# 问卷小筑 · 功能回顾总入口

> 本文件是 devDoc 知识库的总入口和回顾任务看板。
> 所有功能的复盘状态、文档双链、简历价值评级都在此维护。
> 编写和维护规则见 [[CLAUDE]]（项目根目录）。

---

## 项目总目标

通过系统性地回顾已实现功能和学习新功能代码，达成：

1. 重新理解或新学习每个功能的关键代码和设计思路
2. 沉淀为可复习、可支持面试表达的技术文档
3. 补足功能背后的前端基础概念、知识点、八股和教学例子
4. 形成对当前方案的工程判断力，并沉淀可落地的优化思路
5. 将高价值功能的技术描述同步到 [[claude-简历母版]]
6. 每个功能稳定产出四份文档：学习拆解 + 功能复盘 + 概念学习 + 方案评审

### 每个功能对应的文档

- 学习拆解文档：`学习拆解-{功能名}.md`，帮助重新看懂代码
- 功能复盘文档：`功能复盘-{功能名}.md`，帮助准备简历和项目表达
- 概念学习文档：`概念学习-{功能名}.md`，帮助补基础、补知识点、补八股

### 固定执行顺序

1. 输出 `概念学习-{当前功能}.md`
2. 输出 `学习拆解-{当前功能}.md`
3. 输出 `功能复盘-{当前功能}.md`
4. 输出 `方案评审-{当前功能}.md`
5. 更新总入口 / 导航 / 双链
6. 如果有必要，同步更新 [[claude-简历母版]]

## 推荐回顾顺序

基于简历价值和代码回顾难度的综合考量：

1. **SSE 流式生成** — 已完成四文档沉淀，仍然是当前知识库里最值得展开的单点；后续 AI Copilot 工作台就是沿这条流式能力继续升级出来的
2. **AI 对话式问卷工作台** — 已完成四文档沉淀，推荐和 SSE 文档一起看，能更完整地理解“从一次性生成到对话式生成 / 修改”的演进路径
3. **编辑器状态管理** — 核心能力体现，面试必问
4. **物料组件体系** — 抽象设计能力，和编辑器紧密关联，建议连着回顾
5. **拖拽排序** — 可以和编辑器一起回顾，独立性强
6. **移动端答题状态** — 独立模块，回顾成本低

第二批建议在第一批全部完成后再开始，优先做“问卷逻辑编排”（如果有开发计划）或“Lighthouse 性能优化”（如果想快速出成果）。

## 旧功能升级方式

对于已经按旧规则只产出两份文档的功能，按下面方式补齐：

1. 保留原有 `学习拆解` 和 `功能复盘` 文档
2. 新增 `概念学习` 和 `方案评审` 文档
3. 回头补全总入口中的四文档双链
4. 状态从“待补齐”改为“已完成”
5. 如有必要，把新增的优化表达同步进 [[claude-简历母版]]

---

## 后续新增功能登记方式

当有新功能需要纳入回顾体系时：

1. 在上方对应批次中新增一个功能条目，填写完整的属性表格
2. 创建四个双链占位（学习拆解 + 功能复盘 + 概念学习 + 方案评审），标注“待创建”
3. 按 [[CLAUDE]]（项目根目录）中的命名规范命名文档
4. 评估简历价值和优先级，插入到合适的位置
5. 如果是待开发功能，在“证据是否充分”中标注“待开发”
